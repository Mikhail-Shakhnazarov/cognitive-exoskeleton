# Contributing to Cognitive Exoskeleton

Thank you for your interest in contributing! This project welcomes contributions in several forms.

## Ways to Contribute

### 1. Use It and Report Back
- Try the framework in your own work
- Document what worked and what didn't
- Share your experience via issues or discussions

### 2. Improve Documentation
- Fix typos or unclear explanations
- Add examples or clarifications  
- Translate to other languages

### 3. Share Implementations
- Built a writeOS or researchOS implementation?
- Adapted devOS for your context?
- Share it! We can link to it or incorporate it.

### 4. Report Issues
- Found a bug in templates or documentation?
- See a gap in the framework?
- Open an issue with details

### 5. Propose Enhancements
- Have ideas for improvements?
- See missing components?
- Open an issue to discuss before implementing

## Contribution Process

1. **Fork the repository**
2. **Create a feature branch** (`git checkout -b feature/your-feature`)
3. **Make your changes**
4. **Test if applicable** (for code changes)
5. **Commit with clear messages** (`git commit -m 'Add: specific feature'`)
6. **Push to your fork** (`git push origin feature/your-feature`)
7. **Open a Pull Request**

## Style Guidelines

### For Code/Templates
- Keep formatting consistent with existing files
- Comment non-obvious decisions
- Follow existing naming conventions

### For Documentation
- Write in clear, direct prose
- Use examples where helpful
- Link to related sections
- Keep tone professional but not academic-stuffy

### For Theoretical Content
- Cite sources where appropriate
- Be precise with terminology
- Connect to existing framework concepts

Not sure if something is a good contribution? Open an issue to discuss first. No question is too small.

## Code of Conduct

Be respectful, constructive, and kind. We're building tools for thought together.
